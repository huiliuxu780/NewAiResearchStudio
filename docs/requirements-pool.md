# AI 研究平台 - 需求池

## 采集源类型扩展需求

### 需求背景
不同类型的数据源需要不同的采集策略，当前设计需要扩展以支持多种采集模式。

### 需求详情

| 采集类型 | 说明 | 示例 | 技术方案 |
|----------|------|------|----------|
| **搜索关键词** | 通过搜索引擎关键词获取信息 | Google 搜索"通义千问 发布" | SerpAPI / 自建搜索 |
| **整页读取** | 读取单个页面的完整内容 | 官方博客文章、新闻页面 | Crawl4AI 单页抓取 |
| **子页面遍历** | 从列表页进入详情页抓取 | 产品更新日志列表 → 各详情页 | Crawl4AI 多页抓取 |
| **官方号观测** | 监控社交媒体官方账号 | X(Twitter)、微信公众号、微博 | Playwright / 第三方 API |

### 数据模型扩展建议

```python
# Source 模型扩展字段
class SourceType(Enum):
    SEARCH_KEYWORD = "search_keyword"      # 搜索关键词
    SINGLE_PAGE = "single_page"            # 单页面
    MULTI_PAGE = "multi_page"              # 多页面遍历
    SOCIAL_MEDIA = "social_media"          # 社交媒体
    RSS_FEED = "rss_feed"                  # RSS 订阅
    API_ENDPOINT = "api_endpoint"          # API 接口

# 新增字段
class Source:
    # ... 现有字段
    crawl_strategy: str                    # 采集策略
    crawl_config: JSON                     # 采集配置（关键词、选择器等）
    social_platform: str                   # 社交平台（X、微博、微信）
    social_account_id: str                 # 社交账号 ID
```

### 优先级建议

| 优先级 | 类型 | 原因 |
|--------|------|------|
| P0 | 整页读取 | 最基础，覆盖大部分官方文档 |
| P1 | 子页面遍历 | 覆盖更新日志、产品列表 |
| P2 | 搜索关键词 | 需要额外 API，成本较高 |
| P3 | 官方号观测 | 技术复杂度高，可能需要第三方服务 |

### 迭代计划

| Phase | 功能 | 说明 |
|-------|------|------|
| Phase 2 | 整页读取 | Crawl4AI 基础抓取 |
| Phase 2.5 | 子页面遍历 | 列表页 → 详情页 |
| Phase 3 | 搜索关键词 | 接入搜索 API |
| Phase 3.5 | 官方号观测 | 社交媒体监控 |

## 前端体验优化需求

### 需求背景
当前侧边栏有 13 个扁平导航项，超出 Miller's Law（7±2）认知负荷原则，需要重构为分组导航以提升用户体验。

### 详细文档
完整的技术架构和 UI 设计方案已在智能体协作中完成，待后续统一开发实施。

### 6.1 侧边栏导航分组重构

| ID | 功能 | 说明 | 优先级 | 状态 |
|----|------|------|--------|------|
| NAV-001 | 导航分组数据结构 | 新增 NavGroup 和 NavItem 类型定义 | P1 | 待实施 |
| NAV-002 | NavGroup 组件 | 分组容器，管理展开/折叠状态 | P1 | 待实施 |
| NAV-003 | NavItem 组件 | 单个导航项，处理活跃状态和 ComingSoon | P1 | 待实施 |
| NAV-004 | Sidebar 重构 | 简化为容器组件，管理全局折叠状态 | P1 | 待实施 |
| NAV-005 | 分组展开动画 | CSS Grid grid-rows 技巧实现平滑动画 | P1 | 待实施 |
| NAV-006 | 折叠模式适配 | 64px 宽度下图标平铺 + Tooltip 展示 | P1 | 待实施 |
| NAV-007 | ComingSoon 标记 | "即将推出" 胶囊标签视觉表现 | P2 | 待实施 |
| NAV-008 | 响应式适配 | 移动端 Sheet 滑出面板方案 | P2 | 待实施 |

**分组方案**：

| 分组 | 图标 | 包含导航项 | 数量 |
|------|------|------------|------|
| 📊 概览 | LayoutDashboard | Dashboard | 1 |
| 📥 数据采集 | Database | 信息源管理、抓取任务、原始记录 | 3 |
| 🧠 AI 处理 | Brain | 标准化事实、研究结论、提示词管理 | 3 |
| 📋 业务管理 | Package | 模型档案、产品档案(待开发)、研究主题(待开发)、周报中心 | 4 |
| ⚙️ 系统 | Settings | 系统日志、系统设置 | 2 |

**技术架构要点**：

```typescript
// 新增类型定义
interface NavItem {
  key: string;
  label: string;
  href: string;
  icon: React.ReactNode;
  comingSoon?: boolean;  // 标记待开发页面
}

interface NavGroup {
  groupKey: string;
  groupLabel: string;
  groupIcon: React.ReactNode;
  items: NavItem[];
  defaultExpanded?: boolean;
}

// 保留向后兼容
const flatNavItems: NavItem[] = navigationConfig.flatMap(g => g.items);
```

**组件拆分**：

```
components/layout/
  sidebar.tsx          (重构 - 主容器)
  nav-group.tsx        (新增 - 分组容器)
  nav-item.tsx         (新增 - 单个导航项)
```

**交互设计要点**：

| 交互 | 行为 |
|------|------|
| 点击分组标题 | 切换该分组的展开/折叠状态 |
| 点击子项 | 正常路由跳转，不改变分组展开状态 |
| 侧边栏折叠 | 所有子项图标平铺展示，分组标题隐藏 |
| URL 直接访问 | 对应分组自动展开 |
| ComingSoon 项 | 点击显示 Toast 提示，不跳转 |

**UI 设计规范**：

| 元素 | 规格 | Tailwind 类名 |
|------|------|---------------|
| 分组标题 | 11px uppercase, 灰色 | `text-[11px] font-semibold uppercase tracking-wide text-muted-foreground` |
| 子项缩进 | 28px 左侧缩进 | `pl-7` |
| 活跃状态 | 紫色背景 + 左侧装饰线 | `bg-sidebar-accent before:absolute before:left-0 before:w-[2px] before:bg-sidebar-primary` |
| 展开动画 | 200ms 平滑展开 | `grid transition-all duration-200 ease-in-out` |
| ComingSoon | 胶囊标签, 10px | `rounded-full bg-muted/50 px-1.5 py-0.5 text-[10px]` |
| Tooltip | 300ms 延迟, 右侧显示 | `delayDuration={300} side="right"` |

**文件变更清单**：

| 文件 | 操作 | 说明 |
|------|------|------|
| `types/labels.ts` | 编辑 | 新增 NavItem、NavGroup 接口和 navigationConfig |
| `components/layout/sidebar.tsx` | 重构 | 简化为容器组件 |
| `components/layout/nav-group.tsx` | 新增 | 分组容器组件 |
| `components/layout/nav-item.tsx` | 新增 | 单个导航项组件 |

**技术风险与注意事项**：

1. **路由兼容性**：`flatNavItems` 确保现有代码无需修改
2. **动画性能**：使用 CSS Grid `grid-rows` 技巧避免 `height: auto` 布局抖动
3. **状态持久化**：可选使用 localStorage 保存分组展开状态
4. **无障碍访问**：支持键盘 Tab 导航、ARIA 属性、WCAG AA 对比度
