# AI 研究平台 - 信息架构

## 1. 页面层级结构

### 1.1 一级页面总览

| 序号 | 页面名称 | 路由 | 核心功能 |
|------|----------|------|----------|
| 1 | Dashboard | `/` | 总览仪表盘，展示关键指标与最新动态 |
| 2 | 信息源管理 | `/sources` | 信息源配置与管理 |
| 3 | 原始记录 | `/raw-records` | 原始采集记录查看与管理 |
| 4 | 标准化事实 | `/facts` | 结构化事实查看与复核 |
| 5 | 研究结论 | `/insights` | 研究洞察与结论管理 |
| 6 | 模型档案 | `/models` | AI 模型档案库 |
| 7 | 产品档案 | `/products` | AI 产品档案库 |
| 8 | 研究主题 | `/topics` | 研究主题管理 |
| 9 | 周报中心 | `/reports` | 日报/周报/专题报告 |
| 10 | 系统设置 | `/settings` | 系统配置管理 |

### 1.2 页面关系图

```
Dashboard (总览)
    ├── 快速入口 → 各功能页
    └── 数据概览 → 各详情页

信息源管理 → 原始记录 → 标准化事实 → 研究结论
     ↓              ↓              ↓           ↓
   配置          采集结果        抽取结果     分析结果

模型档案 ←── 标准化事实
产品档案 ←── 标准化事实
研究主题 ←── 研究结论
周报中心 ←── 研究结论
```

## 2. 各页面详细设计

### 2.1 Dashboard（仪表盘）

**路由：** `/`

**页面模块：**

| 模块名称 | 展示内容 | 数据来源 |
|----------|----------|----------|
| 今日情报概览 | 今日新增情报数、待复核事实数、最新研究结论数、活跃信息源数 | 统计聚合 |
| 采集任务健康状态 | 采集成功率、失败任务数、最近错误 | 任务状态 |
| 近 7 日情报趋势 | 折线图，展示每日情报数量变化 | 统计聚合 |
| 最新事实列表 | 最近 10 条标准化事实，含公司、主题、重要性 | Fact 表 |
| 最新研究结论列表 | 最近 5 条研究结论，含公司、结论类型、影响层级 | Insight 表 |
| 快速操作入口 | 新增信息源、开始采集、生成周报 | 操作按钮 |

**筛选条件：** 公司筛选（全部/阿里/字节/腾讯）

### 2.2 信息源管理

**路由：** `/sources`

**页面模块：**

| 模块名称 | 展示内容 | 交互方式 |
|----------|----------|----------|
| 来源列表 | 信息源表格，含名称、公司、类型、状态、频率 | 表格展示 |
| 筛选区 | 公司筛选、来源类型筛选、状态筛选 | 下拉筛选 |
| 操作区 | 新增信息源、启用/停用、编辑、删除 | 按钮操作 |
| 来源详情 | 来源配置详情、采集历史、最近记录 | 抽屉/弹窗 |

**表格字段：**

| 字段名 | 字段标识 | 说明 |
|--------|----------|------|
| 来源名称 | name | 信息源名称 |
| 所属公司 | company | 阿里/字节/腾讯 |
| 来源类型 | source_type | 官方文档/博客等 |
| 来源 URL | url | 信息源地址 |
| 状态 | enabled | 启用/停用 |
| 采集频率 | schedule | cron 表达式 |
| 优先级 | priority | P0-P3 |
| 最近采集 | last_crawled_at | 最近采集时间 |
| 操作 | actions | 编辑/删除/查看详情 |

## 4. 交互模式

### 4.1 核心交互模式

| 模式 | 适用场景 |
|------|----------|
| 表格 + 筛选 | 列表数据展示 |
| 抽屉 | 快速查看详情，不离开当前页 |
| 弹窗 | 需要用户确认的操作 |
| 详情页 | 复杂内容的完整展示 |
| 卡片 | 研究结论、主题等视觉化展示 |

### 4.2 状态反馈

- 加载状态：骨架屏 / Loading 动画
- 空状态：空状态插图 + 引导文案
- 错误状态：错误提示 + 重试按钮
- 成功状态：Toast 提示

## 5. 响应式设计

### 5.1 断点定义

| 断点名称 | 宽度范围 | 布局调整 |
|----------|----------|----------|
| 移动端 | < 768px | 单列布局，隐藏侧边栏 |
| 平板 | 768px - 1024px | 可折叠侧边栏 |
| 桌面 | > 1024px | 完整布局 |

### 5.2 Phase 1 范围

Phase 1 优先保证桌面端体验，移动端适配在后续迭代中完成。
